# 🎯 YTNiche Band — Faceless YouTube Niche Research Engine

> **Discover • Analyze • Validate • Build Profitable Faceless YouTube Niches**
> A free/low-cost faceless YouTube niche research web app that uses YouTube data, rule-based analysis, transcript-derived strategy rules, and a modular analysis engine to discover, validate, and bend profitable channel ideas.

---

## 📖 Overview

**YTNiche Band** is a complete niche research toolkit for faceless YouTube creators. It transforms the methodology from the **YouTube Niche Master Prompt v6** — originally designed for Claude AI + NexLev MCP — into a **self-contained, executable web application** with:

- 🔍 **YouTube Data API v3** integration for real channel/video data
- 🧠 **Transcript-derived research rules** encoded as pure TypeScript
- 🗺️ **Niche Bending** engine (4-axis: Category / Format / Language / Audience)
- 📊 **Market stage detection** (Breakout / Late Wave / Saturated / Declining)
- 🚧 **Barrier-to-entry scoring** (Money, Skill, Insider Knowledge, Speed)
- 🌊 **Blue ocean gap detection**
- 🧬 **Script DNA extraction & bending**
- 💰 **Opportunity score calculator** with weighted formula
- 🧪 **Low-cost test plan generator** ($100-$200 budget)
- 🌐 **Multiple YouTube data providers** with automatic fallback
- 🔒 **Safe cookie support** for local-only enriched metadata

## 🚀 Quick Start

### Prerequisites
- **Bun** v1.0+ (for running the project)
- **YouTube Data API v3 Key** (free from Google Cloud Console — optional)

### Setup

```bash
# Install dependencies
bun install

# Start dev server
bun run dev

# Or for CLI analysis tool
bun run analyze
```

### Adding a YouTube API Key (Optional but Recommended)

```bash
# Environment variable (local dev)
echo "YOUTUBE_API_KEY=your_key_here" > .env

# Or via Convex (if using backend)
bunx convex env set YOUTUBE_API_KEY your_key_here
```

> **No-Key Mode:** The app works fully without an API key. Analysis runs with heuristic data and local providers. The YouTube API only adds exact stats when available.

## 🌐 YouTube Data Providers

YTNiche Band uses multiple data sources with automatic fallback. No single provider is required.

### Provider Priority

| Priority | Provider | Key Required | Data Quality | When Used |
|----------|----------|-------------|-------------|-----------|
| 1 | **YouTube API** | ✅ | High (exact stats) | Channel/video analysis |
| 2 | **oEmbed** | ❌ Free | Medium (titles only) | Video metadata |
| 3 | **RSS Feed** | ❌ Free | Medium (uploads only) | Recent uploads |
| 4 | **yt-dlp** (CLI only) | ❌ Local | High (local) | CLI analysis |
| 5 | **Manual** | ❌ Always | Low (heuristic) | Fallback when all others fail |

The provider router automatically falls back through providers until data is found.

### 🔒 Local YouTube Access & Cookies

Cookies are **optional** and used **only** for local yt-dlp access to access age-restricted or member-only content.

> **⚠️ Safety Rules:**
> - Cookies are **local-only credentials**. Never share, upload, or commit them.
> - Cookies are **disabled by default**.
> - Cookie content is **never stored in app state, logs, or displayed in the UI**.
> - `.gitignore` protects `cookies.txt`, `private/`, and all cookie-related files.

#### Enabling Cookie Access (CLI Only)

1. Export cookies from your browser (use a "Get cookies.txt" extension)
2. Save to `./private/cookies.txt`
3. Enable in your environment:
   ```bash
   export ENABLE_COOKIES=true
   export YOUTUBE_COOKIES_PATH=./private/cookies.txt
   export ENABLE_YTDLP=true
   ```

#### Cookie .gitignore Protection

The following are already protected by `.gitignore`:
```gitignore
.env
.env.local
.env.*.local
private/
cookies.txt
*.cookies.txt
*.cookie
```

## 🧠 Transcript Research Engine

The core of YTNiche Band is a **modular, rule-based analysis engine** that encodes patterns from successful faceless YouTube channels into deterministic TypeScript functions. No AI costs, no API dependencies for analysis — only YouTube data for channel/video information.

### Modules

| Module | Purpose | File |
|--------|---------|------|
| **Pyramid** | Classify content complexity tier (bottom/middle/top) | `pyramid.ts` |
| **Barriers** | Score money, skill, insider, speed barriers | `barriers.ts` |
| **Saturation** | Detect market lifecycle stage | `saturation.ts` |
| **Niche Bending** | Generate 4-axis niche bend proposals | `nicheBending.ts` |
| **Script DNA** | Extract and bend video script patterns | `scriptDNA.ts` |
| **Blue Ocean** | Detect format-market gaps | `blueOcean.ts` |
| **Opportunity Score** | Weighted composite score (0-100) | `opportunityScore.ts` |
| **Research Rules** | 14 transcript-derived rules | `researchRules.ts` |

### Opportunity Score Formula

```txt
score =  0.20 × demandScore
       + 0.15 × blueOceanScore
       + 0.15 × barrierScore
       + 0.15 × formatProofScore
       + 0.10 × freshnessScore
       + 0.10 × assetPotentialScore
       + 0.10 × userFitScore
       - 0.15 × saturationRisk
       - 0.10 × policyRisk
```

## 🔧 Workflow

| Phase | Name | What It Does |
|-------|------|-------------|
| **0** | Setup | Register date, select analysis mode (Auto/Channel/Video/Keyword) |
| **1** | Discovery | Resolve input via provider layer, fall back to heuristic |
| **2** | Analysis | Format detection, pyramid classification, barrier scoring, saturation, blue ocean, niche bending, script DNA |
| **3** | Validation | Cross-reference rules, weighted score calculation, recommendation |
| **3.5** | Test Plan | $100-$200 low-cost test plan generation |
| **4** | Full Report | All dimensions in a single comprehensive view |

### Analysis Modes

- **AUTO** — No starting point needed. Uses your creator profile to discover opportunities.
- **CHANNEL** — Analyze a YouTube channel URL or handle.
- **VIDEO** — Analyze a specific YouTube video URL.
- **KEYWORD** — Explore a keyword or topic for niche bending opportunities.

### Creator Profile Fields

| Field | Purpose |
|-------|---------|
| Topics | Subjects you can talk about for hours |
| Interests | Things you're passionate about |
| Skills | Your background and abilities |
| Budget | Monthly content production budget |
| Format | Shorts, Long-form, or Both |
| Production Style | AI slideshow, animation, voiceover, etc. |
| Language | Target language/market |
| Goal | Cashflow, asset resale, portfolio, learning |
| Publishing Speed | Daily, weekly, bi-weekly, etc. |
| Example Channels | Channels you admire or want to emulate |
